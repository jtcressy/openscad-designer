# OpenSCAD Designer

An experimental standalone MCP App for interactive parametric CAD with ChatGPT. It combines OpenSCAD Customizer annotations, a generated parameter form, real OpenSCAD-WASM geometry, and a rotatable Three.js preview.

This repository owns the app, MCP server, deployment work, and release lifecycle independently from any particular OpenSCAD model. The initial prototype was developed in [jtcressy/keychains#1](https://github.com/jtcressy/keychains/pull/1) and imported here as a clean standalone snapshot.

## Intended workflow

1. ChatGPT creates or updates a parametric OpenSCAD document.
2. The app parses Customizer annotations and generates controls.
3. Parameter changes are sent back as complete stateless design snapshots.
4. OpenSCAD-WASM renders the snapshot to STL in the browser.
5. Three.js displays the actual mesh and offers STL/3MF downloads when available.

## Development

```sh
npm install
npm run check
npm run dev
```

The development MCP endpoint is `http://127.0.0.1:3001/mcp` and its health check is `/health`. Set `HOST` or `PORT` to override the listener. To work on the iframe UI without rebuilding the server, use:

```sh
npm run dev:ui
```

To exercise the Cloudflare Worker and Static Assets adapter locally instead:

```sh
npm run dev:cloudflare
```

That endpoint defaults to `http://127.0.0.1:8787/mcp` with the same `/health` route.

The main checks cover the Customizer parser, MCP contracts, STL-to-3MF packaging, and a real OpenSCAD-WASM render whose dimensions are read back from the STL.

## Architecture

See the [architecture overview](docs/architecture.md) for the runtime diagram, MCP tool responsibilities, data lifecycle, independent-user isolation boundary, and backend-rendering decision gate. See the [project plan](docs/project-plan.md) for the proposed pull-request sequence through Cloudflare deployment and plugin publication.

- `src/shared/customizer.ts` parses OpenSCAD Customizer groups, descriptions, ranges, dropdowns, vectors, and the `[Hidden]` boundary.
- `src/ui/` is the MCP App: source editor, generated controls, Three.js orbit viewer, local OpenSCAD worker, and downloads.
- `src/server/` exposes four stateless MCP tools plus the shared app resource through runtime-neutral registrations and a Node development adapter.
- `src/worker.ts` exposes those registrations through Cloudflare's Streamable HTTP handler and loads the built UI from a Static Assets binding.
- `skills/openscad-designer/` is the Codex plugin skill that teaches the model how to use the tools.

Every tool accepts and returns the complete design snapshot. The server therefore keeps no user document state and can run behind a horizontally scaled endpoint.

Many people can independently run isolated iframe sessions. Shared documents, membership, permissions, presence, and real-time co-editing are explicitly outside the product scope. Personal file import, export, or persistence can evolve separately without introducing collaboration semantics.

## Build and deployment

```sh
npm run build:cloudflare
```

The UI remains a self-contained HTML asset so its blob-backed OpenSCAD worker and WASM runtime can be tested in ChatGPT before externalizing any browser code. The Node development build embeds that document in its server output. The Cloudflare build instead stores it under Workers Static Assets and keeps it out of the Worker script; the current dry-run Worker bundle is about 264 KiB gzip.

Preview and production publishing is intentionally GitHub-Actions-only. Configure the environment-scoped Cloudflare secret and variables described in the [deployment guide](docs/deployment.md); the two environments must use different Worker names. Pull requests upload isolated preview versions, while `main` is the only production source. No Cloudflare credential files belong in the repository.

To launch in ChatGPT developer mode, use the HTTPS `/mcp` URL from a preview deployment and test a cube render inside the real ChatGPT sandbox. `.app.json` and `.mcp.json` deliberately contain no deployment-specific IDs or URLs yet.

The current all-in-one MCP resource is about 11.9 MB (3.94 MB gzip), mostly the OpenSCAD WASM runtime, and fits Cloudflare's per-static-asset limit. If ChatGPT rejects it or blocks the inline blob worker/WASM policy, split the renderer into versioned HTTPS assets with CORS and add that exact origin to the resource CSP.

## Output and limitations

- STL is generated by the real OpenSCAD WASM engine, not an approximate browser CSG implementation.
- 3MF is packaged from that rendered STL as a millimeter-scale triangle mesh; it does not yet preserve materials or slicer profiles.
- Anycubic Slicer remains downstream: import the STL or 3MF there for orientation, supports, printer profiles, and G-code.
- This build has only been exercised locally; the first developer-mode render is still a required launch test.
- The embedded OpenSCAD runtime is GPL-2.0-or-later, so this project uses the same license declaration.
